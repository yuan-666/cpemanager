# CPE Manager v0.3.0

This alpha release turns the Flutter app into a denser mobile toolboard and adds the first Fiberhome/烽火 configuration adapter from captured HAR traffic.

## Release Assets

| Asset | Use |
| --- | --- |
| `CPEManager-android-v0.3.0-release.apk` | Recommended Android install package for phone testing. |
| `CPEManager-android-v0.3.0-debug.apk` | Debug Android build for development troubleshooting. |
| `CPEManager-macos-arm64-v0.3.0-app.zip` | macOS Apple Silicon Tkinter desktop app bundle. |
| `CPEManager-web-v0.3.0.zip` | Static Flutter Web/PWA build output. |
| `cpemanager-0.3.0-py3-none-any.whl` | Python package wheel for CLI and desktop entrypoints. |
| `SHA256SUMS.txt` | SHA-256 checksums for all release assets. |

## Highlights

- Reworked the Flutter app into a dark, dense CPE dashboard with Huawei/Fiberhome device selection.
- Added PCC, signal quality, carrier/neighbor, lock-frequency, speed, and raw snapshot workspaces.
- Added Fiberhome `FHTOOLAPIS` support for Auto/LTE/SA/NSA network presets, lock Band, NR lock cell, 4G+5G combined lock cell, clear lock-cell list, and readback of lock state.
- Added TAC decimal, LTE ECI, and NR GCI calculations in the mobile dashboard.
- Hardened host input so `192.168.8.1` and `http://192.168.8.1` both work in Dart clients.
- Ignored raw HAR captures to avoid uploading live `sessionid` values.

## Install Android

Enable developer mode and USB debugging, then run:

```bash
adb install -r CPEManager-android-v0.3.0-release.apk
```

The Android package is `com.cpemanager.app`, version `0.3.0`, and allows HTTP access to the local CPE address `192.168.8.1`.

## Install Python CLI

```bash
python -m pip install cpemanager-0.3.0-py3-none-any.whl
cpemanager --help
cpemanager-desktop --version
```

## Fiberhome Notes

- Current Fiberhome support is alpha and based on captured `POST /api/tmp/FHTOOLAPIS` calls.
- The app currently asks for a manual `sessionid`; the HAR set did not include the login/session acquisition flow.
- The HAR set also did not include live RF signal, traffic statistics, device information, or neighbor-cell status endpoints for Fiberhome.
- `act=1` is treated as LTE and `act=2` as NR based on the captured lock-cell requests; verify this on hardware before relying on automated writes.

## Remaining Limits

- Android release signing is development-grade and not yet Play Store/AppGallery ready.
- No Android phone was connected during local verification, so hardware install/launch remains the next smoke test.
- iOS/macOS Flutter native builds still require full Xcode and CocoaPods on this Mac.
- Windows artifacts must be built on Windows.
- HarmonyOS/OpenHarmony needs a separate OpenHarmony-SIG Flutter SDK or ArkTS validation pass.
