# CPE Manager v0.2.0

This alpha release turns the Flutter track into a runnable mobile app and publishes the first GitHub Release assets.

## Release Assets

| Asset | Use |
| --- | --- |
| `CPEManager-android-v0.2.0-release.apk` | Recommended Android install package for phone testing. |
| `CPEManager-android-v0.2.0-debug.apk` | Debug Android build for development troubleshooting. |
| `CPEManager-macos-arm64-v0.2.0-app.zip` | macOS Apple Silicon Tkinter desktop app bundle. |
| `CPEManager-web-v0.2.0.zip` | Static Flutter Web/PWA build output. |
| `cpemanager-0.2.0-py3-none-any.whl` | Python package wheel for CLI and desktop entrypoints. |
| `SHA256SUMS.txt` | SHA-256 checksums for all release assets. |

## Highlights

- Added a mobile-first Flutter CPE dashboard with connection form, status cards, NR neighbor display, raw snapshot view, and guarded automatic-mode/unlock-all actions.
- Ported the core Huawei CPE login and status-read client to Dart.
- Generated Android, iOS, macOS, Windows, and web platform folders.
- Verified Android APK and Web/PWA builds locally.
- Enabled GitHub Actions at `.github/workflows/desktop-build.yml` after refreshing the GitHub OAuth token with `workflow` scope.
- Bumped Python and Flutter app versions to `0.2.0`.

## Install Android

Enable developer mode and USB debugging, then run:

```bash
adb install -r CPEManager-android-v0.2.0-release.apk
```

The Android package is `com.cpemanager.app`, version `0.2.0`, and allows HTTP access to the local CPE address `192.168.8.1`.

## Install Python CLI

```bash
python -m pip install cpemanager-0.2.0-py3-none-any.whl
cpemanager --help
cpemanager-desktop --version
```

## Notes

- This is still an alpha release. Android release signing is development-grade and not yet Play Store/AppGallery ready.
- No Android phone was connected during local verification, so hardware install/launch remains the next smoke test.
- iOS/macOS Flutter native builds still require full Xcode and CocoaPods on this Mac.
- Windows artifacts must be built on Windows.
- HarmonyOS/OpenHarmony needs a separate OpenHarmony-SIG Flutter SDK or ArkTS validation pass.

## Verification

```bash
JAVA_HOME=/opt/homebrew/opt/openjdk@17 flutter build apk --release
JAVA_HOME=/opt/homebrew/opt/openjdk@17 flutter build apk --debug
JAVA_HOME=/opt/homebrew/opt/openjdk@17 flutter build web
JAVA_HOME=/opt/homebrew/opt/openjdk@17 flutter test
JAVA_HOME=/opt/homebrew/opt/openjdk@17 flutter analyze

conda run -n cpemanager python tools/build_desktop.py --onedir
conda run -n cpemanager cpemanager-desktop --version
```
