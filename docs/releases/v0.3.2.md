# CPE Manager v0.3.2

This local build fixes Fiberhome/烽火 readback 403s and improves the mobile dashboard for phone use.

## Build Assets

| Asset | Use |
| --- | --- |
| `CPEManager-android-v0.3.2-release.apk` | Recommended Android phone test package. |
| `CPEManager-android-v0.3.2-debug.apk` | Debug Android package for troubleshooting. |
| `SHA256SUMS.txt` | Checksums for local v0.3.2 Android assets. |

## Highlights

- Fiberhome `FHTOOLAPIS` POST now sends fixed-length JSON and refreshes `get_refresh_sessionid` before each POST.
- LAN CPE traffic bypasses desktop HTTP proxy settings in Flutter and Python clients.
- Added a read-only Fiberhome smoke test script for login plus `app_get_*` verification.
- Added 5-second mobile auto refresh after the first successful status read.
- Added Simple/Pro display modes for translated field labels or raw source parameters.
- Added SIM information for `UL_AMBR`, `DL_AMBR`, and `QCI`.
- Added RF-quality tiles for `DL_Modulation` and `UL_Modulation`.
- Reworked device selection into a scalable device-profile dropdown.
- Fixed mobile tile overflow on long Fiberhome values and reset scroll position on tab changes.

## Remaining Limits

- Android release signing is development-grade and not yet Play Store/AppGallery ready.
- iOS/macOS Flutter native builds still require full Xcode and CocoaPods on this Mac.
- Windows artifacts must be built on Windows.
- HarmonyOS/OpenHarmony needs a separate OpenHarmony-SIG Flutter SDK or ArkTS validation pass.
